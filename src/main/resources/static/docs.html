RMA Movies API Documentation

A read-only REST API serving 1,000 popular movies for the RMA course at the Faculty of Computing, Union University Belgrade.

  • Base URL: https://rma.finlab.rs
  • Authentication: None required — publicly accessible
  • Response format: JSON (application/json), except the root endpoint which returns plain text
  • HTTP Methods: All endpoints use GET

Image URL Construction

The API returns image paths (e.g., /9cqNxx0GxF0bflZmeSMuL5tnGzr.jpg) rather than full URLs. To display an image, construct the full URL using the configuration from the /config endpoint.

Formula:

{image_base_url}{size}{file_path}

Example: To display a movie poster at width 500:

https://image.tmdb.org/t/p/w500/9cqNxx0GxF0bflZmeSMuL5tnGzr.jpg

The available sizes (from /config) are:

Image TypeAvailable Sizes
Posterw92, w154, w185, w342, w500, w780, original
Backdropw300, w780, w1280, original
Profile (people)w45, w185, h632, original
Logo (companies)w45, w92, w154, w185, w300, w500, original
Tip for mobile apps: Use smaller sizes (e.g., w185 for posters in a list) to save bandwidth, and larger sizes (e.g., w500 or original) for detail screens.

Pagination

Endpoints that return lists of items use a paginated response format. You control pagination with page and page_size query parameters.

PaginatedResponse format:

{
    "page": 1,
    "pageSize": 20,
    "totalItems": 1000,
    "totalPages": 50,
    "items": [ ... ]
}
FieldTypeDescription
pageIntegerThe current page number (1-based).
pageSizeIntegerThe number of items per page.
totalItemsIntegerTotal number of items matching the query.
totalPagesIntegerTotal number of pages available.
itemsArrayThe array of results for the current page.

Defaults: page = 1, page_size = 20. The maximum page_size is 100.

Error Handling

When something goes wrong, the API returns a JSON error response with an appropriate HTTP status code.

Error response format:

{
    "error": "undefined",
    "httpCode": 404,
    "message": "Movie not found: invalidid"
}
FieldTypeDescription
errorStringError type identifier.
httpCodeIntegerThe HTTP status code.
messageStringA human-readable error message.
descriptionString?Additional details about the error (optional).
suggestionString?A hint on how to fix the request (optional).

Common HTTP status codes:

CodeMeaningExample
400Bad RequestInvalid query parameter (e.g., page_size=200).
404Not FoundMovie, person, or collection with the given ID does not exist.
500Internal Server ErrorUnexpected server-side failure.
