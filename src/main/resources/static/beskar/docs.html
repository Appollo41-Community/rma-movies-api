Beskar API Documentation

An astronomy content API serving 11,200 NASA APOD entries (1995–2026) with posts, reactions, comments, collections, and categories. Named after the precious Mandalorian armor metal.

  • Base URL: https://rma.finlab.rs
  • Path prefix: All endpoints are under /beskar/
  • Authentication: None required — nickname-based identity for write operations
  • Response format: JSON (application/json)
  • HTTP Methods: GET, POST, PUT, DELETE

Pagination

Endpoints that return lists of items use a paginated response format. You control pagination with page and page_size query parameters.

PaginatedResponse format:

{
    "page": 1,
    "pageSize": 20,
    "totalItems": 11200,
    "totalPages": 560,
    "items": [ ... ]
}
FieldTypeDescription
pageIntegerThe current page number (1-based).
pageSizeIntegerThe number of items per page.
totalItemsIntegerTotal number of items matching the query.
totalPagesIntegerTotal number of pages available.
itemsArrayThe array of results for the current page.

Defaults: page = 1, page_size = 20. The maximum page_size is 100.

Error Handling

When something goes wrong, the API returns a JSON error response with an appropriate HTTP status code.

Error response format:

{
    "error": "undefined",
    "httpCode": 404,
    "message": "Post not found: 99999"
}
FieldTypeDescription
errorStringError type identifier.
httpCodeIntegerThe HTTP status code.
messageStringA human-readable error message.
descriptionString?Additional details about the error (optional).
suggestionString?A hint on how to fix the request (optional).

Common HTTP status codes:

CodeMeaningExample
400Bad RequestInvalid query parameter (e.g., page_size=200).
404Not FoundPost, collection, or comment with the given ID does not exist.
409ConflictDuplicate reaction — user has already liked/disliked this post.
500Internal Server ErrorUnexpected server-side failure.

GET /beskar/posts Try it →

Returns a paginated list of posts. Supports searching, filtering by media type, category, and date range, as well as sorting.

Query Parameters

NameTypeRequiredDescriptionDefault
pageIntegerOptionalPage number (must be >= 1).1
page_sizeIntegerOptionalItems per page (1-100).20
queryStringOptionalSearch posts by title and description (case-insensitive substring match).—
media_typeStringOptionalFilter by media type: image or video.—
category_idIntegerOptionalFilter by category ID (see /beskar/categories).—
start_dateStringOptionalFilter posts on or after this date (YYYY-MM-DD).—
end_dateStringOptionalFilter posts on or before this date (YYYY-MM-DD).—
sort_byStringOptionalSort field. One of: date, title, likes, dislikes, comments_count.date
sort_orderStringOptionalSort direction: asc or desc.desc

Response: PaginatedResponse<PostListItem>

Each item in the items array has this structure:

FieldTypeDescription
idIntegerUnique post identifier.
titleStringPost title.
descriptionStringTruncated description (~150 characters).
mediaTypeStringMedia type: "image" or "video".
imageUrlString?Standard resolution image URL (null for videos).
videoUrlString?Video embed URL (null for images).
thumbnailUrlString?Thumbnail URL (primarily for videos).
mediaAuthorStringAuthor/credit for the media.
dateStringOriginal APOD date (YYYY-MM-DD).
likesIntegerTotal like count.
dislikesIntegerTotal dislike count.
commentsCountIntegerTotal number of comments.

Reactions

POST /beskar/posts/{id}/like

Adds a like to a post from the given nickname. If the user has already disliked the post, the dislike is automatically removed.

Path Parameters

NameTypeDescription
idIntegerThe unique post ID.

Request Body

{
    "nickname": "astro_fan_42"
}

Response: 200 OK

Returns the updated post as a Post object.

Error: 409 Conflict

Returned if the user has already liked this post.

{
    "error": "undefined",
    "httpCode": 409,
    "message": "User astro_fan_42 has already liked post 100"
}
